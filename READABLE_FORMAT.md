# 播客逐字稿阅读版 · 格式规范 v1（定稿）

本规范定义"阅读版"的整理标准,供 readable skill / agent 严格遵循。它与《日报格式规范》
目标相反:**日报是浓缩提炼;阅读版是保真去噪**。阅读版不得压缩信息量。

行业里这种处理叫 **clean read(净读版)**:保留全部实义内容与说话人归属,只去掉口语噪音、
修正标点与识别错字、做利于扫读的分段。它不是逐字稿(保留所有 um/ah、false start),
也不是编辑稿(改写润色、重组段落、加小标题)。我们要的就是中间这一档。

---

## 一、四条硬目标(对应你的需求)

1. **保留说话人**:每次换人另起一段,说话人标注一致、不混用。
2. **保留时间戳**:每个发言块标段首时间戳;长发言块按节奏补中间时间戳。
3. **去口水词**:删无意义语气/填充词(见第三节词表),拿不准就保留。
4. **优化阅读体验**:修正明显同音/识别错字、补全标点、合理分段。

绝对禁止:总结、概括、提炼要点、删任何实质信息、改写句意、调发言顺序、跨人合并、
加带有归纳性质的小标题、把内容改成要点列表。

## 二、允许 vs 禁止(一表说清)

| 操作 | 允许? | 说明 |
| --- | --- | --- |
| 删纯语气词(嗯、呃、啊…) | ✅ | 见第三节词表;拿不准保留 |
| 删填充词(那个、就是说…) | ⚠️ 看语境 | 仅当它无实义时;作指代/强调时必须保留 |
| 删口吃/重复开头(我我我→我) | ✅ | 仅删机械重复,不删强调性重复 |
| 修同音错字(在/再、做/作…) | ✅ | 仅高置信、上下文明确时 |
| 修专有名词识别错 | ⚠️ | 把握大才改;不确定保留原样,可标 `[?]` |
| 补/改标点 | ✅ | 让句子可读,不改语气强弱 |
| 分段(长发言拆成自然段) | ✅ | 不改顺序、不改内容 |
| 合并破碎短句为通顺句 | ⚠️ | 仅同一句被 ASR 切碎时拼回;不跨语义合并 |
| 修正明显问答错配(短答被并进提问块) | ✅ 高置信 | 见第四B节;仅"一问一答被并进同一块"时拆分归还,拿不准不动 |
| 总结 / 提炼要点 | ❌ | |
| 删冗余但有实义的内容 | ❌ | 重复表达只要有信息就保留 |
| 改写润色措辞 | ❌ | 保留说话人原话风格 |
| 加归纳性小标题 / 转要点列表 | ❌ | |
| 跨说话人合并、调顺序 | ❌ | |

口诀:**改"形"不改"义",去"噪"不去"料"。拿不准,一律保留。**

## 三、口水词 / 填充词参考表

**几乎总可删(纯语气噪音)**:嗯、呃、唉、哎、啊(句首或独立时)、欸、哦(语气)、嗯嗯、
啊这个(连读填充)、哈(非笑声语气)。**叠词形式**(啊啊、哦哦、哎哎、嗯嗯、嗯啊)同样几乎总可删。

> **强约束(欠清洗)**:净读版正文里不应再残留 `嗯 / 呃 / 唉 / 哎 / 欸` 任何一个,也不应出现
> `啊啊 / 哦哦 / 哎哎` 这类叠词语气。单字 `啊 / 哦 / 哈` 作句末语气词时是实义,保留(如"好啊""对哦")。
> `verify_readable.py` 会逐块标出这些残留(见第七节"审计告警")。

**看语境删(无实义时才删)**:那个、这个(作填充而非指代时)、就、就是、就是说、然后
(纯连接冗余时)、其实(冗余时)、的话、对吧、你知道吧、怎么说呢、我觉得(高频冗余时)。

**必须保留**:
- 作指代的"那个/这个"(那个项目、这个数据)。
- 有实义的"对/对对/没错"(表态、回应)。
- 说话人明显在强调的重复(如"非常非常重要")。
- 专业术语、人名、产品名、英文词、数字。

## 四、错别字 / 识别错修正规则

- 只修**高置信**错误:同音字(在/再、做/作、的/得/地、帐/账)、明显音转错词。
- 专有名词:把握大才改(如"特斯拉"被识别成"特撕拉");不确定保留 ASR 原文,可在词后加
  `[?]` 提示人工复核,**不要凭空猜一个名字**。
- **术语词典(`glossary.json`)**:英文专名/术语(尤其模型名、产品名、公司名)最容易被中文 ASR 听坏,
  且前后拼法不一致(如 `cloudcode/cloudcold`=Claude Code、`URUYX/URUX`=UI/UX、`majourney`=Midjourney)。
  以本仓库 `glossary.json` 为准:`canonical` 是规范写法白名单,`corrections` 是已知错拼→规范的映射。
  整理时**逐条套用 corrections**,并把术语归一到 canonical;发现新的高置信错拼就补进词典。词典里没有、
  又拿不准的英文词,**必须标 `[?]` 或补进词典**,不能把乱码专名当确定内容留着。校验脚本会加载词典把关(见第七节)。
- 不要"纠正"说话人故意的口语/方言用法。
- 数字、单位、年份按上下文还原(如"二零二六年"→"2026 年"可,但不改数值)。
- 英文大小写只做轻量修正:常见英文词、品牌名和词组不应出现单词内部随机大写
  (如 `OPEnAI`、`customer exPErience`、`one PErson company`);但节目里的专有缩写和
  术语保持原样,如 `FDE`、`FDPM`、`PE`、`API`、`LLMs`、`SaaS`。

## 四A、独立回应词(backchannel)处理 — 默认规则

对话里常有只含一个回应词的独立发言块(对方说话时的"嗯""啊""哦")。默认处理:

- **删除纯语气 backchannel**:整块只有"嗯/呃/啊/哦/哎/唉/欸/呐/嗯嗯/啊啊/哦哦/哎哎"这类无实义回应的,**整块删掉**
  (它不承载信息,保留只会打断阅读)。这是 backchannel 的唯一例外——允许删整块。
- **保留有实义的短回应**:整块是"对/对对/没错/是的/不是/不对/没有"等**表态**的,**保留**为一行
  (它是真实的回应内容,删了会丢信息)。
- 校验脚本据此放行:被删的纯 backchannel 块不计入"块数/顺序必须一致"的校验,不报错。

> 这是默认值,可整体切换为"全部保留(纯 backchannel 也留成一行)"。

## 四B、说话人问答错配修正(高置信)

diarization 在"快问快答""短促一问一答"的节奏下,常把那 0.x 秒的极短回答(如"三十七岁")
错并进上一个人的提问块里。这是**原始转写就有的边界错误**,阅读版可以修,但只在高置信时修。

触发场景(满足其一):
- 上一块以明确**问句**结尾(请问…?、你的…呢?),紧跟一个短答案;或
- 处于**快问快答 / 连续短问答**段落:主持人块里出现了"本应由嘉宾本人给出的事实性答案"
  (年龄、学历、星座、收入等关于嘉宾自身的信息),且该内容在语义上与嘉宾紧邻发言连续。
  注意:快问快答里主持人的提示常是省略式("毕业院校是…?"),ASR 可能不带问号、还把答案
  并进来,所以**不要求一定有问号**。

处理:在"提示词"与"答案"之间切开——答案归还给嘉宾(可与其紧邻发言合并);主持人保留
提示词并补成问句(如"毕业院校是"→"毕业院校是?")。**只移动、补标点,不增删任何实义词。**

约束(防止乱改归属):
- 拿不准就**不动**;只在快问快答段或明确问答场景里做,不做全局重排。
- 只有当答案"自证属于嘉宾本人"(是嘉宾的个人事实)且与其上下文连续时才拆;若主持人确在
  陈述/念资料,保留原样。
- 每次拆分/改归属要可追溯:产出加脚注或写进校验日志,便于人工复核。

## 五、排版与 Markdown 格式

### 文件头
```
# <单集标题>

> 节目：<节目名> ｜ 发布：<日期> ｜ 时长：<时长>
> 原始链接：<小宇宙链接>
> 整理方式：阅读版(去口水词 + 修错字 + 排版;保留全部实质内容,未总结/删减/合并)
> 逐字稿原文：<逐字稿文件名>.md

---
```

### 正文
- 每次换人 = 一个新发言块。说话人用**加粗**,后接段首时间戳(用行内代码包裹):

```
**主持人**  `00:00`
正文第一段……

**嘉宾**  `01:12`
正文第一段……

正文第二段(同一发言人长段落拆出来的自然段)……  `02:40`
```

- 说话人标注全程一致(主持人/嘉宾,或真实姓名),不在中途改叫法。
- **说话人映射(每集可手动指定)**:diarization 给的是 spk 0/1/2…,且在快问快答等段落易错位。
  支持每集填一份映射(spk → 真实姓名/角色,如 `0=主持人, 1=课代表`),阅读版按映射出名字;
  未指定时回退为"说话人 1/2…"。映射在 dashboard 里填一次,或写进单集记录。
- **正文高置信说话人映射**:若节目简介、开场白、`【主播】/【嘉宾】`名单、主持人点名
  ("嘉宾B 介绍一下自己吧")、或自我介绍("我是嘉宾C")能稳定证明某个声轨对应真实姓名,
  阅读版应将该声轨全篇统一替换为真实姓名。只允许一对一稳定映射;同一声轨证据冲突、或只能知道
  候选名单但不能确定归属时,保留"说话人 X"并标 `[?]`,不要猜。
- **说话人映射 sidecar**:凡阅读版把 `说话人 X` 替换为真实姓名/角色,必须在阅读版同目录写入
  同名 `<阅读版文件名去掉 .md>.speaker-map.json`,内容为 JSON 对象,如
  `{"说话人 1": "主持人A", "说话人 2": "嘉宾B"}`。若 diarization 把同一主持人拆成多个声轨,
  可高置信映射为同一个名字,如 `{"说话人 2": "主持人A", "说话人 3": "主持人A"}`。
- **混剪/孤儿声轨**:片头新闻混剪、片尾曲、插入的影视片段等,会以多个只出现一两次的"说话人 N"
  出现。它们是**真实的不同音源**,不是 diarization 把主播切错——**保留为独立声轨,不要并进主播,
  也不要强行起名**。校验脚本会把"只出现 ≤2 次且未映射的说话人 N"标为提示(混剪 or 切错,人工判断)。
  只有当确属同一主播被切成多轨(上下文连续、同一段对话)时,才在 speaker-map 里高置信合并。
- **语义断裂的乱码段**:整句已被 ASR 打碎、无法高置信复原的(典型如片头混剪),不要臆造内容,
  在该句末尾标 `[?]` 表示整句存疑;乱码密集时校验脚本会用 `[?]` 密度与乱码启发式提示回听。
- 时间戳格式:不足 1 小时用 `mm:ss`,超过用 `hh:mm:ss`。
- **分段标准**:同一人的长发言按语义/换气拆成自然段,每段约 **150–300 汉字**;
  发言块超过约 3 分钟时,在新自然段段尾或段首补一个中间时间戳。
- 不使用归纳性二级标题。若节目本身有清晰的环节切换(如"进入问答环节"是说话人原话),
  可保留为正文,不要自造小标题。

### 可选(默认关闭)
- 顶部"时间轴目录":仅按时间戳 + 说话人列出锚点,**不写内容概括**。需要时再开。

## 六、Before / After 示例

**逐字层(FunASR 原始)**
```
**说话人 1**
[00:00:03.200 - 00:00:11.500]
嗯，那个，我们今天呢，就是说，主要想聊一聊这个，呃，AI 在播客里面的应用，对吧。

**说话人 1**
[00:00:11.500 - 00:00:15.000]
就是它它它其实已经在很多场景里面用起来了。
```

**阅读版(本规范)**
```
**主持人**  `00:00`
我们今天主要想聊一聊 AI 在播客里面的应用。它其实已经在很多场景里面用起来了。
```
说明:删了"嗯/那个/呢/就是说/呃/对吧"等口水词、修了"它它它"口吃;**没有**删任何实质信息
("主要想聊 AI 在播客的应用""已经在很多场景用起来了"全部保留),没有总结成一句话。

## 七、与校验脚本的衔接

`scripts/verify_readable.py` 按本规范把关。它分两层:

**硬闸门(失败即退出码非 0,保真优先):**
- 按说话人发言块对齐逐字层 vs 阅读版,块数与顺序必须一致。
- 实义字符保留率(剔除空白、标点、第三节口水词后的中文字符)每块 ≥ 0.85,低于则报错。
- 说话人错配(未在 speaker-map 里登记的)、英文单词内部随机大写,报错。
- 这道闸门是为了兜住"清洗变成总结"的风险——保真优先于好看。

**审计告警**:默认只提示、不改退出码。告警分两档,报告里 `!` 是"blocking"(加 `--strict` 会判失败)、
`~` 是"advisory"(即使 `--strict` 也只提示,因为多为源质量信号、误报率较高)。

blocking(`--strict` 下失败):
- **欠清洗**:正文里残留的 `嗯/呃/唉/哎/欸/啊啊/哦哦/哎哎` 等高置信口水词,逐块列出。
  (注意:保留率只防"删多",对"没删干净"是透明的,所以单设这道告警。)
- **术语词典错字**:`glossary.json` 的 `corrections` 里已知的错拼(如 `cloudcode`→Claude Code、
  `URUYX`→UI/UX、`实诗`→史诗)如果在阅读版里还没改,逐块标出——这是逼着改写层套词典。
- **反向保留率**:阅读版里对不回逐字层的内容(可能是新增/改写/幻觉),逐块提示。保留率只防删、
  不防增,这道反向闸门补上"禁止加内容"。
- **结构 lint**:正文混入"听不清/无法辨认"等编辑批注(应改用 `[?]`)、时间戳缺失或格式不对。

advisory(仅提示,供人工/改写层排查):
- **`[?]` 密度**:每千实义字的 `[?]` 数过高,提示源 ASR 质量差、需回听。
- **乱码启发式**:同字 ×3、超长无标点中文串、零散单字母混入等,提示语义断裂段。
- **孤儿/混剪声轨**:只出现 ≤2 次且未映射的"说话人 N"(片头/片尾混剪 or diarization 切错,人工判断)。
- **词典外英文词**:既不是常见英文、又不在 `glossary.json` 里的小写英文词(如 `mistholes`、`feeble`),
  很可能是被听坏的专名或该补进词典的新词。
- **拼写变体不一致**:低频且编辑距离很近的英文词聚成一簇(如 `majourney/midjourney`、`devon/davon`),
  多半是同一个词的多种错拼。

用法:`python scripts/verify_readable.py <逐字稿>.md <阅读版>.md --speaker-map-file <…>.speaker-map.json`
(自动加载仓库 `glossary.json` 与同名 `<阅读版>.glossary.json`;可用 `--glossary` 再加)。
把 blocking 告警当硬性要求时追加 `--strict`。
